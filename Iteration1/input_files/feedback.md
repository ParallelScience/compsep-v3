The current analysis failed due to a fundamental misunderstanding of the Wiener Filter (WF) application in the context of extreme dynamic range. The "numerical collapse" reported is not a failure of floating-point precision, but a direct consequence of applying a global linear filter to a signal that is spatially sparse and non-Gaussian.

**Critical Weaknesses:**
1. **Misapplication of the Wiener Filter:** The WF is an optimal estimator for Gaussian random fields. The tSZ signal is highly non-Gaussian and spatially sparse (clusters occupy <1% of the sky). By using the global power spectrum ($P_\ell$) to define the signal covariance $S_\ell$, you have effectively "smeared" the cluster signal across the entire map. The filter treats the cluster as a diffuse, low-amplitude background component, leading to the observed suppression.
2. **Failure to Standardize:** You correctly identified the $10^{15}$ variance disparity but failed to implement the necessary pre-conditioning. Applying a linear filter to raw maps where the CIB variance is 15 orders of magnitude larger than the tSZ variance will always result in the filter weights being dominated by the foregrounds, effectively nulling the tSZ signal.
3. **Redundant "Constrained" Benchmark:** The constrained MWF benchmark was a waste of resources. If you have the ground truth to compute the cross-power for a specific patch, you have already solved the problem. It provides no insight into the performance of a blind detection pipeline.

**Actionable Recommendations for Future Iterations:**
1. **Abandon Global Linear Filtering:** Do not attempt to "fix" the MWF with more regularization. It is fundamentally ill-suited for sparse, non-Gaussian signal extraction.
2. **Adopt a Multi-Scale/Patch-Based Approach:** If you must use linear methods, move to a **Matched Filter (MF)** approach rather than a Wiener Filter. The MF is designed to detect specific spatial templates (cluster profiles) at specific locations. It is far more robust to foregrounds because it operates in the spatial domain (or filtered Fourier domain) on a per-cluster basis, rather than attempting to reconstruct the entire map globally.
3. **Implement Pre-whitening:** Before any component separation, you must "whiten" the data. Use the noise covariance $N_\ell$ to transform the maps such that the noise is identity-distributed. This is standard practice in CMB analysis (e.g., the `ilc` or `milca` algorithms) and is essential to prevent the dominant foregrounds from dictating the filter weights.
4. **Shift to Object Detection:** Given the sparsity of the tSZ signal, treat this as an **object detection problem** rather than a map-reconstruction problem. Use a simple CNN or a U-Net with a weighted loss function (as suggested in the dataset description) to identify cluster candidates. This avoids the need to reconstruct the "empty" sky pixels, which are currently driving your numerical collapse.
5. **Validation:** Stop using global SNR thresholds. Because the noise is heterogeneous (varying CIB and SO noise), your detection threshold must be **local**. Calculate the local noise variance in a sliding window around each candidate to determine if a peak is a true detection or a noise fluctuation.

**Summary:** The project has learned that global linear estimators are insufficient for this dataset. Future iterations should focus on local, template-based matched filtering or non-linear machine learning architectures that can handle the sparsity of the tSZ signal.
